🔧 Supabase RLS 错误修复

问题描述：如果你遇到 "row-level security policy" 错误，这是因为Supabase表启用了行级安全策略。

📋 修复步骤

方案1: 禁用RLS（推荐）

适合个人使用的Chrome扩展，简单有效：

-- 禁用行级安全策略 ALTER TABLE quick_nav_data DISABLE ROW LEVEL SECURITY;

方案2: 配置RLS策略

如果你想保持安全策略启用：

-- 启用行级安全策略 ALTER TABLE quick_nav_data ENABLE ROW LEVEL SECURITY; -- 创建允许所有操作的策略 CREATE POLICY "Enable all access for authenticated users" ON quick_nav_data FOR ALL USING (true) WITH CHECK (true);

方案3: 重新创建表

如果上述方案都不行，可以重新创建表：

-- 删除现有表（注意：会丢失数据） DROP TABLE IF EXISTS quick_nav_data; -- 重新创建表 CREATE TABLE quick_nav_data ( id SERIAL PRIMARY KEY, user_id TEXT NOT NULL UNIQUE, data JSONB NOT NULL, created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(), updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() ); -- 禁用RLS ALTER TABLE quick_nav_data DISABLE ROW LEVEL SECURITY; -- 创建索引 CREATE INDEX idx_quick_nav_data_user_id ON quick_nav_data(user_id);

🚀 执行步骤

  1. 打开 Supabase Dashboard
  2. 选择你的项目
  3. 点击左侧菜单的 SQL Editor
  4. 粘贴上面复制的SQL命令
  5. 点击 Run 执行
  6. 返回Chrome扩展重新测试同步功能

🔍 验证修复

修复后，在Chrome扩展中：

  1. 点击同步按钮
  2. 点击"测试连接"
  3. 如果成功，点击"启用云端同步"
  4. 应该能正常同步数据
💡 提示：对于个人使用的Chrome扩展，禁用RLS（方案1）是最简单有效的解决方案。
